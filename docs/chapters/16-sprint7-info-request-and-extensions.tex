\section{Bối cảnh}

Sprint 6 đã wire controller cho 10 module portal hiện có. Sprint 7 bổ sung:

\begin{itemize}[leftmargin=*]
    \item \textbf{Module mới} \texttt{wujia\_portal\_info\_request} --- cửa hàng nhượng quyền gửi yêu cầu cập nhật thông tin (địa chỉ, SĐT, người đại diện...) qua portal, HQ duyệt qua chatter backend (Phase 1, BA sheet \texttt{2. FE - Portal} row ``Cập nhật thông tin'').

\end{itemize}

\subsection{Security}

\begin{itemize}[leftmargin=*]
    \item \texttt{ir.model.access.csv}: portal (\texttt{group\_portal}) read/write/create (không unlink), internal user full.
    \item Record rule portal read: \texttt{[('franchise\_id', 'in', user.\_get\_accessible\_franchise\_ids())]}.
    \item Record rule portal write: \texttt{[('created\_by\_user\_id', '=', user.id)]} --- chỉ tự sửa của mình.
    \item Internal: rule \texttt{[(1, '=', 1)]} full access.
\end{itemize}

\subsection{Views}

5 file XML: list (filter state + request\_type + pager), form (AJAX old\_value), detail (readonly + cancel button form POST), backend (tree/form/search + 3 action buttons start\_review/approve/reject), sidenav\_inherit (menu ``Cập nhật thông tin'' vào \texttt{wujia\_portal\_layout.layout\_sidenav}).

\textbf{Performance per route}:
\begin{itemize}[leftmargin=*]
    \item Attachment download: ACL check \texttt{accessible\_franchise\_ids} trước; stream qua \texttt{ir.binary.\_get\_stream\_from} (không load full bytes vào memory); \texttt{Cache-Control: private, no-cache}.
    \item PDF/XLSX: dùng Odoo report engine native + xlsxwriter \texttt{in\_memory=True}.
    \item ICS: text response simple, gen string < 1KB, không cần DB write.
\end{itemize}
