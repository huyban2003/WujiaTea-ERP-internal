\section{Bối cảnh}

BA giao bộ \textbf{design-system source-of-truth} trong \texttt{WujiaTea/UI/}:
\texttt{mobile\_source\_ui\_v2\_3} (tokens \texttt{--wj-*}, foundation component SVG + spec MD,

cho desktop, \texttt{--wj-pc-*}). Mục tiêu: chuẩn hóa token + component cho toàn portal về
đúng BA, \textbf{giao diện chính KHÔNG đổi}, làm nền sạch trước khi dựng UI mới.

\textbf{Phát hiện trung tâm:} code desktop đang dùng brand \texttt{\#22A9DE} (code cũ coi
\texttt{\#28A9DF} là ``typo'' rồi giữ \texttt{\#22A9DE}). Nhưng BA source \textbf{khóa
\texttt{\#28A9DF} và CẤM \texttt{\#22A9DE}} (ghi rõ ở README + Decision Log + Registry + QA).
Mobile (\texttt{--wujia-mshell-*} / \texttt{--wujia-morder-*}) đã dùng \texttt{\#28A9DF} \(\rightarrow\)
desktop và mobile \textbf{không đồng nhất brand}. Đây là lệch lớn nhất cần gỡ.

\textbf{Quyết định user (đã chốt):} (1) độ sâu = \textbf{token + component} (không rework
từng màn); (2) \textbf{adopt BA toàn bộ} kể cả các giá trị trước đây cố ý override (page bg,
danger, success); (3) \textbf{shared + mobile trước}, PC/desktop reconcile theo
\texttt{pc\_source\_ui\_v1\_4} tách đợt sau; (4) ghi mọi phần tách-đợt-sau vào compact-summary.
