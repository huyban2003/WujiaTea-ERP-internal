\section{(A) Controller batch-centric, gộp một nguồn desktop + mobile}

\texttt{portal\_delivery\_list} \textbf{bỏ hẳn} khối query picking-centric cũ; query batch (đã có từ S24) giờ nuôi \textbf{cả desktop lẫn mobile}. Mỗi dict batch thêm keys PC (\textbf{không đụng} keys mobile \texttt{id/name/label/modifier/departure/orders}): \texttt{pc\_label}/\texttt{pc\_modifier}, \texttt{departure\_full} (``dd/mm/yyyy · HH:MM''), \texttt{vehicle} (``name · driver''), \texttt{plate}, \texttt{updated} (``HH:MM'').

Bổ sung: (1) param \texttt{q} tìm mã chuyến/mã SO (\texttt{name ilike} \(|\) \texttt{picking\_ids.sale\_id.name ilike}); (2) helper \texttt{\_chip\_counts} đếm 4 nhóm (Tất cả/Đang giao/Sắp giao/Đã giao) qua \texttt{search\_count} trên base domain franchise+date (\textbf{không} status, \textbf{không} keyword) --- field \texttt{delivery\_batch\_status} có index, 4 count rẻ, an toàn 1500 user; (3) helper \texttt{\_page\_numbers} (copy pattern history) cho pager numbered; (4) \texttt{view\_state} + \texttt{?\_preview=} giờ \textbf{drive luôn block desktop} (5 state). \texttt{m\_pager} thêm \texttt{page\_total}/\texttt{page\_nums}/\texttt{offset}/\texttt{count} (superset, mobile chỉ đọc subset cũ).

\texttt{portal\_delivery\_detail} thêm: \texttt{pc\_badge}, \texttt{pc\_departure}/\texttt{pc\_updated}/\texttt{pc\_orders} (reuse helper), \texttt{so\_names} (list SO unique cho chip), \texttt{tl\_steps}/\texttt{tl\_times} (timeline 3 bước done/active/inactive suy từ \texttt{actual\_departure}+\texttt{delivery\_batch\_status}), \texttt{src\_location} (\texttt{own\_pickings[0].location\_id.complete\_name}), \texttt{store\_label}. Bỏ 2 map legacy \texttt{BATCH\_STATUS\_LABELS}/\texttt{PICKING\_STATUS\_LABELS} (\texttt{wujia-badge-*}).

  \item \textbf{Loading skeleton = UI-only} (server-render, không fetch client) --- build pixel-perfect, xem qua \texttt{?\_preview=loading}, chừa cho AJAX. Error wire thật qua try/except.
  \item \textbf{Date filter để trống + chips số đếm là filter thật} (không hardcode ``7 ngày tới'' như Figma) --- đồng bộ hành vi mobile S24, subtitle ghi ``N chuyến · Mới nhất trước'' đúng thực tế.
  \item \textbf{Giữ native date input} (house pattern history/notification) thay vì widget dd/mm/yyyy custom trong Figma --- tránh over-engineer, browser vi-VN tự render đúng locale.
  \item \textbf{Title 32px + crumb-trên-title scoped riêng} delivery (house PC dùng 26px + crumb-dưới) --- theo Figma nhưng không regress history/notification.
\end{itemize}
