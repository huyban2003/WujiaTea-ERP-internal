\chapter{Sprint 35 --- Dashboard BOD Step 1/6: port Dashboard Ninja v18 $\to$ Odoo 19 (2026-07-16)}

\section{Bối cảnh}

BOD cần dashboard backend (không đụng portal 1500 user). User sở hữu Dashboard Ninja
(Ksolves, OPL-1) bản \textbf{v16} tại \texttt{vietthuong/tools/} (core 16.0.1.7.4 +
\texttt{ks\_dn\_advance} 16.0.2.0.5 + \texttt{ks\_dn\_formula} --- formula engine + pivot
tự viết) và bản \textbf{v18 core} 18.0.1.0.1; bản v19 chính hãng (\$592) không mua.
Hướng đi: tự migrate, ưu tiên \emph{tính năng chạy trước, giao diện sau}.

Discovery (5 agent đọc song song + adversarial review) chốt: \textbf{v18 = base}
--- đã OWL 2 full rewrite (v16 core là AbstractAction 4000+ dòng chết từ Odoo 17),
toàn bộ chart thống nhất amCharts 5, SCSS design system mới; gap 18$\to$19 nhỏ hơn 16$\to$19
rất nhiều. 12/13 method Python mà advance/formula hook vẫn còn nguyên trong v18 core
$\Rightarrow$ Python của 2 module đó sẽ port gần nguyên vẹn (Step 2--3), chỉ JS phải viết
lại theo kiến trúc component v18.

Đây là \textbf{workstream đa session} (6 step); nguồn sự thật riêng:
\texttt{docs/dashboard-migration-plan.md} (step table, discriminator rules, 22 gotchas),
kickoff qua skill mới \texttt{/wujia-dashboard}. Sprint log vẫn ghi như thường.

\section{Lý do / Trade-off}

Chọn v18 làm base thay vì migrate thẳng v16 để khỏi viết lại 4000+ dòng AbstractAction
đã chết ở Odoo 17+ --- công việc Ksolves đã làm ở v18; đổi lại mất các field styling
Chart.js chi tiết của v16 (amCharts 5 style khác). amCharts để eager thay vì lazy bundle:
backend chỉ vài chục user BOD, portal 1500 user không load
\texttt{web.assets\_backend} nên không ảnh hưởng hiệu năng portal. Còn lại: Step 2
(advance subset), Step 3 (formula+pivot --- tài sản chính), Step 4 (test suite),
Step 5 (dọn \texttt{read\_group} deprecated --- vẫn chạy đúng ở 19), Step 6 (reskin).

\section{HOTFIX Step 1 (Sprint 38 --- 2026-07-16): prop \texttt{ks\_speak} sót sau strip AI}

Sau khi Step 1 đóng, mở board có item ở chế độ \textbf{debug} (\texttt{?debug=1}) văng
\texttt{OwlError: Invalid props for component `Ksdashboardtile': `ks\_speak' is missing}.
Không phải thiếu module hay package --- là bug thuần front-end (OWL props) sót lại khi strip AI.

\begin{itemize}
  \item \textbf{Gốc rễ:} 3 component tile/kpi/graph còn khai prop
    \texttt{ks\_speak:\{type:Function, Optional:true\}} kèm nút ``voice-button'' (text-to-speech
    của AI đã strip). \texttt{Optional} viết HOA \emph{không} phải key OWL (OWL chỉ nhận
    \texttt{optional} thường) $\to$ OWL coi prop là \textbf{bắt buộc}. Bản v18 gốc luôn truyền
    \texttt{ks\_speak} từ template AI nên không lộ; bản wj strip AI $\to$ grid
    \texttt{ks\_dashboard\_item\_template} không truyền $\to$ validate fail, sập cả board.
  \item \textbf{Chỉ nổ ở debug mode:} OWL chỉ chạy \texttt{validateProps} khi
    \texttt{env.debug || session.test\_mode} (\texttt{web/static/src/env.js:236}); chế độ user
    thường bỏ qua validate nên board vẫn render $\Rightarrow$ verify Step 1 ``0 console error''
    chạy ở chế độ thường đã lọt bug này (chỉ lộ khi bật Developer Mode).
  \item \textbf{Fix:} xoá 3 khai prop \texttt{ks\_speak} + 12 block voice-button
    (tile 7, kpi 3, graph 1, list 1; 148 dòng). \texttt{aiAudioRef} JS refs còn lại null-safe
    (\texttt{?.}) nên vô hại; \texttt{.ks\_speaker} CSS + 1 dòng XML comment để lại (dead, harmless).
  \item \textbf{Verify:} \texttt{-u wj\_ks\_dashboard\_ninja} RC=0 / 0 ERROR log, 4 XML well-formed,
    server restart sạch trên :8019, \texttt{test\_sprint9} 7/7; bump manifest $\to$ 19.0.1.0.1
    để auto-deploy regen asset bundle.
\end{itemize}

\textbf{Bài học:} verify gate browser phải bật \texttt{?debug=1} \emph{và} mở board-detail có
item, không chỉ trang landing; sai hoa/thường \texttt{Optional} là latent bug toàn code
