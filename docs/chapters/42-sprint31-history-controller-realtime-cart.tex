\section{Part A --- Controller Lịch sử đơn hàng (CT-024 + CT-025)}

\subsection{Quyết định nghiệp vụ (nguồn: chat BA, FINAL)}

\begin{itemize}
\item \textbf{Phạm vi}: lấy \emph{tất cả} \texttt{sale.order} của cửa hàng hiện tại (cả đơn
      portal lẫn đơn nhập tay backend --- NQ có cả đơn bù/trả). \textbf{Loại đơn huỷ}
      (\texttt{state = cancel}) khỏi lịch sử.
\item \textbf{Cửa hàng}: strict current store qua \texttt{get\_active\_franchise\_id()}; chưa
      chọn $\rightarrow$ báo ``Không xác định được cửa hàng đang thao tác''. Không tin store id
      từ FE. Detail search lại theo store $\rightarrow$ chống IDOR, không lộ tồn tại đơn cửa
      hàng khác (cùng 1 message).
\item \textbf{Ngày}: list hiển thị/lọc/sort theo \texttt{create\_date} (NQ quan tâm giờ tạo);
      detail hiện cả \texttt{create\_date} (ngày tạo) và \texttt{date\_order} (ngày xác nhận).
\item \textbf{Người đặt}: đơn portal $\rightarrow$ \texttt{portal\_requester\_user\_id.name};
      đơn backend $\rightarrow$ nhãn \textbf{``Ngô Gia tạo đơn''} (không lộ \texttt{create\_uid}
      nội bộ lên portal).
\item \textbf{Giá đã gồm thuế}: thành tiền dòng = \texttt{price\_total}; đơn giá =
      \texttt{price\_total / qty} (sau chiết khấu, đã thuế) --- \emph{không} dùng
      \texttt{price\_subtotal}. Tổng chính thức = \texttt{amount\_total}.
\item \textbf{Batch}: 1 SO $\leftrightarrow$ 1 batch qua \texttt{sale.order.batch\_id}; trạng thái
      \texttt{delivery\_batch\_status}, ghi chú \texttt{delivery\_note}, giờ khởi hành
      \texttt{actual\_departure} \texttt{or} \texttt{planned\_departure}.
\item \textbf{Số sản phẩm} = số dòng \texttt{display\_type = False} (loại section/note).
\end{itemize}

\section{Part B --- Realtime giỏ hàng cross-session (bus.bus)}

\subsection{Kiến trúc}

Server đã publish \texttt{bus.bus} event \texttt{wujia\_cart\_changed} trên channel
\texttt{wujia.franchise\_<id>} mọi mutation giỏ (Sprint 30). Sprint này thêm tầng client
\textbf{Interaction} \texttt{portal\_cart\_sync.js} (selector \texttt{\#wj-cart-sync}, pattern y hệt
\texttt{franchise\_realtime.js} --- ADR-006):

\begin{itemize}
\item \textbf{Subscribe} bus $\rightarrow$ event $\rightarrow$ fetch lại \textbf{partial giỏ}
      server-render qua route mới \texttt{/portal/order/cart/fragment} (dùng chung QWeb, không dựng
      row bằng JS) $\rightarrow$ \textbf{swap} DOM không reload.
\item \textbf{Event delegation} trên \texttt{document} cho stepper/xoá/ghi chú $\rightarrow$ sống sót
      qua fragment swap (không rebind). Số liệu qty/giá lấy từ server (pricelist có bậc).
\item \textbf{Bỏ mọi \texttt{location.reload()}}: thêm/sửa/xoá giỏ cập nhật DOM live cùng-tab;
      catalog badge + floatbar + header badge reconcile từ \texttt{qty\_map} trong state.
\end{itemize}

Tách partial \texttt{mcart\_panel} (mobile) + \texttt{pc\_cart\_panel} self-contained; floatbar
mobile luôn render (ẩn \texttt{d-none} khi trống) để JS toggle. Gỡ \texttt{portal\_order\_pc.js}
(handler chuyển hết sang delegation). \texttt{wujia\_portal\_sale} 19.0.4.0.0.

\subsection{Gotcha môi trường --- WebSocket dev}

Bus realtime chạy qua \textbf{WebSocket} phục vụ bởi gevent worker (\texttt{gevent\_port}), production
proxy qua nginx \texttt{/websocket}. Server \textbf{dev local} (Werkzeug, \texttt{workers=0}, không
gevent worker) \textbf{không phục vụ WebSocket} $\rightarrow$ mọi bus realtime của dự án (kể cả
chuông thông báo ADR-006) không truyền tin ở local. \textbf{Đây là giới hạn môi trường, không phải
lỗi code}. Verify client bằng cách gọi thẳng callback \texttt{onCartChanged} mà WebSocket sẽ kích
hoạt $\rightarrow$ chuỗi reconcile (fetch fragment + apply DOM) đúng; chỉ ống transport bị chặn ở local.
