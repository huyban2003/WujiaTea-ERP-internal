\chapter{Sprint 23 --- Chốt sổ UI mobile theo Figma: spacing + typography (batch 1) (2026-06-25)}

\section{Bối cảnh}

Sau $\sim$12 sprint dựng mobile từng trang rải rác, spacing/typography giữa các trang \textbf{trôi không
đồng nhất}: top-pad 14/16/24/28px, title 20--25px, search 30/38/42/44px. User báo 2 điểm cụ thể:
(1) \texttt{/portal/notification} --- title + search ``sát phía trên'', cảm giác chật; (2)
\texttt{/portal/order} --- title ``Sản phẩm'' \textbf{xa header} + chữ \textbf{hơi nhỏ}. BA đã có số đo
Figma cụ thể (file copy \texttt{aoeiDYlg6vlhJZg2w6Q7o5}) cho padding + size title từng trang $\rightarrow$
mục tiêu đợt này: đưa code khớp Figma để \textbf{chốt sổ} mobile. Quyết định với user: (a) title
\textbf{theo Figma từng trang} (chấp nhận 22/24/25/26 khác nhau --- đúng phân cấp BA); (b) khi số Figma
mâu thuẫn cảm giác nhìn $\rightarrow$ \textbf{bám Figma trước, soi 390px cùng user}; (c) Delivery
\textbf{tạm hoãn} (chưa có frame Figma).

\section{Gốc rễ: hai cơ chế top-padding}

Phát hiện qua đọc code: trang mobile chia 2 nhóm top-pad khác nhau ---
\begin{itemize}
  \item \textbf{BlankShell} (\texttt{.wujia-mpage}: Thông báo, Hỗ trợ, Đổi trả, Kiến thức, Tài khoản) render
        NGOÀI \texttt{.content-wrapper}, top-pad $=$ \texttt{--wujia-mshell-content-gap} $= 14$px.
  \item \textbf{Trang cũ trong \texttt{.content-wrapper}} (Đặt hàng, Giỏ, Lịch sử) dính
        \texttt{html body .content .content-wrapper \{ padding-top: var(--wujia-page-content-top) /*24*/ \}};
        override mobile chỉ tắt \texttt{margin-top}, KHÔNG tắt \texttt{padding-top} $\rightarrow$ $24 + 4$px
        titlerow $= \sim 28$px. Đây là lý do ``Sản phẩm'' xa header.
\end{itemize}

Fix top-pad cho nhóm content-wrapper bằng \texttt{:has()} (đã dùng sẵn ở \texttt{\_components.css}), scope
trong \texttt{@media (max-width: 991.98px)} nên \textbf{desktop giữ nguyên 24px}:
\begin{lstlisting}
@media (max-width: 991.98px) {
  .content-wrapper:has(.wujia-morder) { padding-top: 8px !important; }  /* 8+4 = 12 (Figma) */
  .content-wrapper:has(.wujia-mcart) { padding-top: 8px !important; }
  .content-wrapper:has(.wujia-mhist) { padding-top: 1px !important; }   /* 1+4 = 5 (Figma) */
}
\end{lstlisting}

\section{Lý do / Trade-off}

\begin{itemize}
  \item \textbf{\texttt{:has()} thay vì sửa token \texttt{--wujia-page-content-top}}: token đó dùng chung
        desktop + home $\rightarrow$ bump = side-effect toàn portal. \texttt{:has()} scope theo wrapper +
        trong media mobile $\rightarrow$ blast radius $= 0$ trên desktop.
  \item \textbf{Title Đổi trả scoped}: \texttt{.wujia-mticket-h1} dùng chung Hỗ trợ + Đổi trả; Figma muốn
        26 (Đổi trả) vs 22 (Hỗ trợ) $\rightarrow$ override SCOPED \texttt{.wujia-mreturn}, KHÔNG đụng class gốc.
  \item \textbf{Thông báo top-pad 10px}: Figma (10) NHỎ hơn code cũ (14) --- ngược cảm giác ``chật'' của user.
        User chọn ``bám Figma + soi cùng anh'' $\rightarrow$ áp 10px, chốt thị giác ở bước review 390px;
        nguyên nhân ``chật'' thật sự nằm ở dải store-strip phía trên (cần soi xác nhận).
  \item \textbf{Top-pad override} \texttt{.wujia-mpage.wujia-mX} (specificity $0,0,2,0$) thắng \texttt{.wujia-mpage}
        ($0,0,1,0$) bất kể load-order $\rightarrow$ override chắc chắn không phụ thuộc thứ tự file CSS.
\end{itemize}
